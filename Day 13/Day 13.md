## 🧠 Reflexión

### ¿Qué fue fácil?
- Usar `print()` para debugging es la técnica más básica y directa
- Reproducir bugs es simple cuando sabes cómo activarlos

### ¿Qué fue difícil?
- Aprender a usar el debugger de PyCharm requiere práctica
- A veces es difícil "pensar como la computadora" (play computer)

### ¿Qué aprendí?
- **Describe the Problem**: Hablar del problema en voz alta ayuda a encontrar la solución (rubber duck debugging)
- **Reproduce the Bug**: Si puedes reproducir el error, puedes arreglarlo
- **Play Computer**: Ejecutar el código mentalmente línea por línea
- **Fix the Errors**: Leer los mensajes de error cuidadosamente (SyntaxError, TypeError, etc.)
- **Use Print**: Insertar prints estratégicos para ver valores de variables
- **Use a Debugger**: Usar breakpoints y step-through para inspeccionar el código en ejecución

### Técnicas de debugging que aprendí:
1. 🦆 **Rubber Duck Debugging** - Explicar el problema en voz alta
2. 🐛 **Print Debugging** - Insertar prints para rastrear valores
3. 🔍 **Debugger** - Usar herramientas como PyCharm debugger
4. 🎯 **Reproduce First** - Hacer que el error ocurra consistentemente
5. 🧠 **Play Computer** - Ejecutar el código mentalmente
6. 📖 **Read Error Messages** - Los errores te dicen exactamente qué está mal
